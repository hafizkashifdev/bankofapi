Getting Started - IndicativeFXRatesAPI API

This is a Getting Started in Sandbox page, please consult this link for a more comprehensive documentation and potential next steps.

Using the IndicativeFXRatesAPI API from an application

This section aims at giving you a comprehensive step by step tutorial to guide you through the process of configuring the usage and executing the API IndicativeFXRatesAPI (v1) by foreign-exchange.org

Assumptions

  • You have already registered an App in your Team
  • Or you use the demo one

Steps

1. Retrieve access token

To invoke the api foreign-exchange.org/IndicativeFXRatesAPI, you'll need to pass an access-token identifying your App: simply replace CLIENT_ID and CLIENT_SECRET in the curl below to retrieve a valid one:


Request: 
curl -k -X POST \
  https://ob.sandbox.natwest.com/token \
  -H 'Content-Type: application/x-www-form-urlencoded' \
  -d 'grant_type=client_credentials&client_id=CLIENT_ID&client_secret=CLIENT_SECRET'
Response: 
HTTP/1.1 200
status: 200
content-type: application/json; charset=utf-8
 
{
    "token_type": "Bearer",
    "access_token": "Ci5A6huK2OD0d6...kVwLE9FuHwHGCf6xo5",
    "expires_in": 300000
}


Your access-token is the value of the access_token attribute in the Response.
in the example above it would be Ci5A6huK2OD0d6...kVwLE9FuHwHGCf6xo5


2. Invoke relay/prototype endpoints of IndicativeFXRatesAPI (v1) by foreign-exchange.org

Finally you can perform api calls for which the user approved the consent. You can invoke the relay/prototype api foreign-exchange.org/IndicativeFXRatesAPI/v1 by replacing the ACCESS_TOKEN and using the documented methods/resources in the curl below:

Request: 
curl -k -X <available method> \
  https://ob.sandbox.natwest.com/zerocode/foreign-exchange.org/IndicativeFXRatesAPI/v1/<resource exposed by the api> \
  -H 'Authorization: Bearer ACCESS_TOKEN' \
  -H 'Accept: application/json'
  • Make sure the Accept header value matches the one advertised by the configured API specification.
  • The response will be the response from the configured API backend and will vary for different APIs.
