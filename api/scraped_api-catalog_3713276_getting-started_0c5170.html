Getting Started - Balance Reporting Webhooks

Key Sandbox Endpoints

  • Well-known: https://api.sandbox.natwest.com/.well-known/openid-configuration
  • Token: https://ob.sandbox.natwest.com/token
  • Resource:
    • https://ob.sandbox.natwest.com/open-banking/v3.1/event-subscriptions
    • https://ob.sandbox.natwest.com/open-banking/v3.1/event-subscriptions/{subscriptionId}
    • https://ob.sandbox.natwest.com/open-banking/v3.1/events
    • https://ob.sandbox.natwest.com/zerocode/bankofapis.com/balance-reporting/1.0.0/
    • https://ob.sandbox.natwest.com/zerocode/bankofapis.com/balance-reporting/1.0.0/resend

Assumptions

  • You have already registered an App in your Team
  • Or you use the demo one

Postman

If you are a Postman user, you can easily explore and run the steps of this tutorial using our predefined collection and environment files:
Note that in Postman you have to go in Preferences or Settings > General menu
  • in REQUEST column, make sure that Ssl certificate verification is set to OFF (or make sure that the OpenBanking Pre-Production Issuing CA is in your trusted store)
  • in HEADERS column, make sure that Automatically follow redirects is set to ON

Download and import in Postman
  • Download your bankofapis.com/balance-reporting Postman collection
  • Download your Postman environment files per Application below:
    • demo-app-3c216362-f073-42f2-8985-af4031365d94
A personal callbackUrl will need to be added to the environment file you use. This will need to be owned by you and capable of receiving HTTP POST requests.

Receiving Event Notifications

Events are made available via webhooks which will post them to a specified url. They can also be retrieved using aggregated polling.
Before you can receive events, you must first set up your event subscription.
Sandbox limitations:
  • The only event type supported in Sandbox is Discretionary.BalanceReport:OBIE.
  • Hard-coded example events are used - repeated triggering of events will create duplicate events.
  • The resend-correlation-id would normally be added as a header on resent events but this is not possible in Sandbox.
  • Payload.SourceData.ConsentIds would normally be required in the data body of balance-reporting subscriptions request but these are not returned in the Sandbox response.

Retrieve access token

To make a call to sandbox, you'll need to pass an access-token identifying your App: simply replace CLIENT_ID and CLIENT_SECRET in the curl below to retrieve a valid one:

Request 

curl -k -X POST \
  https://ob.sandbox.natwest.com/token \
  -H 'Content-Type: application/x-www-form-urlencoded' \
  -d 'grant_type=client_credentials&client_id=CLIENT_ID&client_secret=CLIENT_SECRET&scope=accounts'

Response 

HTTP/1.1 200
status: 200
content-type: application/json; charset=utf-8
 
{
    "token_type": "Bearer",
    "access_token": "iMVB9i7ZChEe9GV...XeOOCD1opOA2IAULQ",
    "expires_in": 7200,
    "scope": "accounts"
}
Your access-token is the value of the access_token attribute in the Response.
in the example above it would be iMVB9i7ZChEe9GV...XeOOCD1opOA2IAULQ

Create Event Subscription

Now you can create a subscription by replacing the ACCESS_TOKEN in the curl below.
You also need to replace the CALLBACK_URL with a URL of your choice or remove that line.
You do not need to include the Payload object but this would be required in actual balance-reporting subscriptions.

Request 

curl -k -X POST \
  https://ob.sandbox.natwest.com/open-banking/v3.1/event-subscriptions \
  -H 'Authorization: Bearer ACCESS_TOKEN' \
  -H 'Content-Type: application/json' \
  -H 'x-jws-signature: DUMMY_SIG' \
  -d '{
  "Data": {
    "Version": "3.1",
    "CallbackUrl": CALLBACK_URL,
    "EventTypes": [
      "Discretionary.BalanceReport:OBIE"
    ],
    "Payload": {
      "SourceData": {
          "ConsentIds": [
              "CONSENT_ID"
          ]
        } 
      }
    }
  }

Response 

HTTP/1.1 201
status: 201
content-type: application/json;charset=utf-8

{
    "Data": {
        "EventSubscriptionId": "224f1617-9209-4c3a-94d4-94e5629f45ba",
        "Version": "3.1",
        "CallbackUrl": "https://my-domain:1234/callback",
        "EventTypes": [
            "Discretionary.BalanceReport:OBIE"
        ]
    },
    "Links": {
        "Self": "https://ob.sandbox.natwest.com/open-banking/v3.1/event-subscriptions/224f1617-9209-4c3a-94d4-94e5629f45ba"
    }
}

Trigger An Event

To simulate a balance change and trigger a balance reporting event, just replace the ACCESS_TOKEN in the curl below.
If your subscription has a callback url specified, the event will automatically be posted to the specified callback url.
Notes:
  • You must have a service running to accept the events. They will be sent as HTTP POST requests.
  • Each request will trigger a single event. Repeated requests will trigger a new event containing the same stub data.
  • The request will contain a 'x-fapi-interaction-id' header with a unique randomly generated value.
  • Event notifications must be acknowledged with a 2xx (Success) HTTP response.

Request 

curl -k -X GET \
  
https://ob.sandbox.natwest.com/zerocode/bankofapis.com/balance-reporting/1.0.0/ \
  -H 'Authorization: Bearer ACCESS_TOKEN'

Polling Events

If a subscription is created without a CallbackUrl value, events will be stored and can be consumed using aggregated polling by replacing the ACCESS_TOKEN in the curl below:
Notes:
  • maxEvents: max value is 100 (default)
  • returnImmediately: only 'true' (default) is supported

Request 

curl -k -X POST \
  https://ob.sandbox.natwest.com/open-banking/v3.1/events \
  -H 'Authorization: Bearer ACCESS_TOKEN' \
  -H 'Content-Type: application/json' \
  -d '{
  "maxEvents": 100,
  "returnImmediately": true
}

Response 

HTTP/1.1 200
status: 200
content-type: application/json;charset=utf-8

{
    "moreAvailable": false,
    "sets": {
        "77f88e43-c575-423b-8046-a66191e82e2b": "eyJhbG...NEysZ4",
        "0849fc04-bc4c-463f-805d-0813587c4e94": "eyJhbG...Qssw5c",
        "c5948e79-3004-461f-b7b7-ebf8a406b686": "eyJhbG...9kogfI"
    }
}

Resend Events

If a subscription is created with a CallbackUrl value, but the event fails to send, the event will be stored. The sending process for any failed event can be retried using the resend endpoint. This can be simulated by replacing the ACCESS_TOKEN in the curl below:

Request 

curl -k -X POST \
  https://ob.sandbox.natwest.com/zerocode/bankofapis.com/balance-reporting/1.0.0/resend \
  -H 'Authorization: Bearer ACCESS_TOKEN' \

Response 

HTTP/1.1 200
status: 200
content-type: application/json;charset=utf-8

{
    "count": 1,
    "resend-correlation-id": "1aa8bedf-ca02-4e96-9699-36e5948b5429"
}
Note: In production the resend-correlation-id will be added as a header on outgoing resent webhook events.
